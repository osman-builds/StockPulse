Local Deployment

To run the complete production-ready application including database replicas, Redis caching, Nginx load balancers, and JWT portals, run locally via Docker Compose:

# Clone the repository
git clone https://github.com/osman-builds/StockPulse.git
cd StockPulse
# Build and launch multi-service stack
docker compose up --build -d

The services will spin up and start listening on port **8000** (handled by Nginx). The database automatically seeds with sandbox users and items for frictionless initial tests.

System Features Summary
  • ✨
    Premium Standalone Authentication Screen
    A dedicated login, sign-up, and OTP portal styled with a high-end glowing glassmorphism aesthetic. Auto-redirects sessions depending on role metadata.
  • 🛡️
    Zero-Configuration Sandbox OTP
    If SMTP host details are omitted from the environment settings, the application runs in local Mock mode, displaying registration verification codes instantly on the screen with an Autofill action.
  • 📷
    Real-Time Camera Scan & Live Preview
    Integrated device camera capturing allows users to point their lenses at real QR/barcodes to populate details, dynamically resolving inventory levels, safety thresholds, and safety alerts in real-time.
  • 🎛️
    Multi-Service Docker Deployment Architecture
    Routes traffic using Nginx to spread API requests across two independent backend replica containers backed by a persistent PostgreSQL database instance and high-performance Redis cache rings.
